Cricket FFT

Cricket FFT is a Fast Fourier Transform library designed specifically for iOS and Android native development.  It is optimized for ARM devices, using NEON instructions when available, and can also be easily built for Windows and OS X.

Using Cricket FFT

What follows is a very brief introduction to using Cricket FFT.  For more detailed API documentation, have a look at the header file (inc/ckfft/ckfft.h).  There is also a simple working example program in src/example. 

First, create a CkFftContext object with a call to CkFftInit().  For example:

#include "ckfft/ckfft.h"

// Create a context for forward or inverse FFTs with n=2048 or less
CkFftContext* context = CkFftInit(2048, CkFftDirection_Both, NULL, NULL)
Typically you will create one context when your program starts up that will be used for all FFTs.  If you pass in NULL for the last two arguments, CkFftInit() will allocate its own memory for the context.

Most likely, you will want to do a forward FFT on real input data; for example:

// input is an array of 2048 floats;
// output is an array of 1025 CkFftComplex structs.
CkFftRealForward(context, 2048, input, output);
Note that for an FFT on N real values, only N/2+1 complex values are calculated, rather than N; the remaining values are simply complex conjugates of the first values.

You can also perform the inverse transform:

// output is the array of 1025 CkFftComplex structs from the previous example;
// output2 is an array of 2048 floats;
// tmpBuf is an array of 1025 CkFftComplex structs.
CkFftRealInverse(context, 2048, output, output2, tmpBuf);
Finally, destroy your context when you no longer need it (when your program exits, for example):

CkFftShutdown(context);

Building with Cricket FFT

Take a look at the project in src/example for an example of how to build Cricket FFT and link to it on the various platforms.

For iOS or OS X, add the Xcode project in src/ckfft/ios or src/ckfft/macos to your workspace.  In the build settings for your executable project, add the inc directory to your header search path; in the build phases for your executable project, add libckfft_ios.a or libckfft_macos.a to the "Link Binary With Libraries" phase.

For Windows, there are project files for both Visual Studio 2010 and Visual Studio 2012 in src/ckfft/win.  Add the appropriate project to your solution, add the inc directory to your header search path, add the appropriate library directory to your library search path, and add ckfft.lib to your link line.

For the Android NDK, an Android.mk file is included in src/ckfft/android which defines it as a "prebuilt" library.  Use $(call import-module,ckfft) in your shared library's Android.mk to import it.
